Claude Code Custom Skill

The /gap-analysis
Skill

A reusable command that turns raw evidence into a structured, human-reviewable report

✅

Trigger when asked for…

Gap analysis · current-state assessment · remediation plan · technical debt review · options analysis · before/after design comparison

🚫

Don't trigger for…

A standalone diagram → use generating-mermaid-diagrams
A targeted code fix → use the relevant generating-* skill

📋

Required inputs

Topic / scope · raw issues or evidence · audience · report title

⚙️

Install once, use everywhere

Copy to ~/.claude/commands/ for global access, or into a project's .agents/skills/ for team sharing via git

/gap-analysis

Tab-completable in any Claude Code session after install

Phase Flow & Output

Three gated phases,
one offline HTML report

Human review checkpoints between each phase — edit the markdown, then say "proceed"

Phase 1

Discovery

Ingest evidence. Classify each gap by severity (P1 / P2 / P3) and business impact. Write gaps.md.

✋ Stop — review & edit gaps.md
→
Phase 2

Design

Short-term fixes (≤ 4 weeks) + long-term target state. Solution comparison cards per problem area. Write design.md.

✋ Stop — review & edit design.md
→
Phase 3

Report

Generate a single self-contained HTML file from the approved markdown files. No internet required to open.

📄 Output: docs/[topic]/[title].html
TAB 1

Current State — severity table + problem-map SVG

TAB 2

Solution Options — short-term vs long-term cards

TAB 3

Short-Term Design — remediation steps + before/after code

TAB 4

Long-Term Design — target architecture + migration sequence

Opens from file:// — no CDN needed
Under 800 KB
Tailwind CSS · inline SVG · system fonts
